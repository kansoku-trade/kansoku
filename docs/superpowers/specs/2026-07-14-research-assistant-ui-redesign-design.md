# 研究 AI 助手面板重构：统一对话流

日期：2026-07-14
范围：纯前端（`apps/web`），后端 / contract / service 不动。

## 背景与问题

研究页「AI 助手」标签当前从上到下垂直堆五块：研究刷新卡片（常驻，含进度和完整报告）、待审阅修改区、聊天记录、输入框、修改历史区。确认的问题：

1. 东西太多太挤，层次不清。
2. 研究刷新是偶尔用的功能，却常驻顶部占一大块。
3. 刷新报告和聊天是两套独立展示，像两个功能硬拼。
4. 修改历史沉底，待审阅修改出现时又冒出一块，位置不稳定。

## 设计决策（已确认）

- 布局方向：**统一对话流**——整个面板就是一条聊天时间线 + 底部输入区。
- 报告展示：**分节折叠卡**，留在流内，摘要常显，四个小节各自折叠。
- 报告留存：流里**只显示最近一次**刷新任务，不做任务历史。
- 提案审阅：**沿用现有弹窗**（ResearchEditReview 原样复用）。
- 修改历史：移进面板右上角图标，弹窗查看。

## 面板结构

```
┌─────────────────────────────┐
│ (header 区)          🕘 历史 │  ← 修改历史入口，弹窗
├─────────────────────────────┤
│                             │
│   对话流（时间混排三类条目）    │
│                             │
├─────────────────────────────┤
│ [N 条修改待审阅]（条件显示）    │
│ [输入框……] [🔄 刷新] [➤ 发送] │
└─────────────────────────────┘
```

## 对话流条目

按时间戳混排三类条目：

### 1. 聊天气泡

现有 `ConversationTranscript` 的行，不变。

### 2. 研究刷新卡（只有最近一次任务）

按任务 `startedAt` 插入时间线，同一张卡随任务状态切换形态：

- **运行中**：五阶段进度（制定计划 → 核查文档 → 检查市场 → 综合证据 → 生成提案）+ 当前活动文案 + 开始时间。复用现有 `TaskProgress` 的内容。
- **完成**：原地变成**分节折叠报告卡**：
  - 摘要常显（`report.summary`）+ 统计行（N 条结论 · N 条风险 · N 个证据源）。
  - 四个小节各自独立折叠，默认收起、显示条数：主要结论 / 风险与反证 / 待验证问题 / 证据来源。折叠交互沿用现有 `EvidenceList` 的 `details` 模式，小节内部渲染（置信度标签、引用编号、来源链接）复用现有 `FindingList` / `EvidenceList` 的展示。
  - 报告产生了提案时，卡内保留「已生成修改提案」提示。
- **失败 / 中止**：错误态卡片（现有 `research-refresh-terminal` 的内容）。

### 3. 修改提案卡

- **待审阅**：醒目卡片——摘要 + N 处修改 + 「待审阅」标记，点击打开审阅弹窗。
- **已处理**（已应用 / 已拒绝 / 已撤销 / 已失效）：同样式但弱化，显示状态与处理时间，点击仍可打开弹窗（查看或撤销）。
- 流内显示的提案数量与现有逻辑一致：全部 pending + 最近 6 条已处理，按 `createdAt` / `resolvedAt` 插入时间线。

## 输入区

- 输入框 + 「刷新研究」按钮 + 发送按钮。
- 刷新任务运行中：刷新按钮变为停止按钮；聊天发送不受影响。
- 聊天生成中：发送按钮变为停止（现状不变）。
- 存在待审阅提案且其卡片不在视野内时，输入框上方显示固定提示条「N 条修改待审阅」，点击滚动到最近一条待审阅卡片。

## 修改历史

- 面板右上角小图标（History），点开弹窗。
- 弹窗内容即现有历史列表：摘要 + 状态 + 时间，点击条目进审阅弹窗。
- 面板底部常驻历史区删除。

## 空态

没有任何聊天和刷新记录时：

- 一句介绍文案（现有 `emptyText`）+ 三个建议问题（现状保留）。
- 加一行提示刷新研究的入口（指向输入区按钮），替代原先的大块介绍卡。

## 增补（2026-07-14 第二轮，实测截图反馈后确认）

首轮上线后确认三处细节问题：双大块标签难看且和下面的孤行断裂、空态文案贴顶、输入区「刷新」按钮窄到文字折行。确认方案：

1. **去掉「关联资料 / AI 助手」双标签，侧栏合并成一页。**
   - 顶部一行：左侧「AI 助手」标题文字，右侧两个小图标动作——刷新研究（RefreshCw，运行中变停止方块，禁用态跟随 pending）和修改历史（History，开现有弹窗）。
   - 关联资料降级成对话流上方一张可折叠小卡（`details`，默认收起）：收起时显示「关联资料 · N 个标的 · N 条相关记录」；展开显示关联标的 chips 和相关记录列表（沿用现有渲染）；全局记录显示现有的无标的文案。固定在头部行下方，不随对话流滚动。
   - `ResearchContext` 的 tab 状态删除；关联资料数据以 props 传给 `ResearchAssistant`（或由其上层组合），`research-context-tabs` 样式删除。
2. **输入区只剩输入框 + 发送**：刷新按钮从输入区移除（入口在顶部图标）。
3. **空态垂直居中**：介绍文案和建议问题在对话流可用高度内垂直居中，不再贴顶。
4. 跟进债一并处理：`ResearchAssistant.tsx` 已达 499 行，本轮必须把 `ResearchEditReview` + `openEditReview` 抽到 sibling 文件。
