# 收费路线图（探索稿）

日期：2026-07-18
状态：方向探索，非实施承诺
前置：`2026-07-18-open-core-boundary-rework-design.md`（phase 3 边界重划）

## 当前收费边界（phase 3 之后）

免费：全部核心功能 + 自带 key 的 AI（复评、对话、macro 过滤、AI 设置基座）+ 研究库浏览。
收费：个股自动跟踪（scheduler）、深度研究 deep-dive、研究库 AI（刷新/编辑/研究对话）。

## 候选方向（四类）

### 一、后台自动化——「不开电脑它也在干活」

与现有付费三项同一逻辑，收费正当性最强（持续占用计算、持续烧 token、用户不在场也产生价值）：

- 盘中自动警报：突破、放量、MACD 背驰、跌破持有计划触发线时通知。警报必须绑定纪律规则（触发线、trade-gate），不绑定裸价格波动——避免做成「付费买焦虑」（见 TD-NOISE-01）。
- 财报季自动化：盯 finance-calendar，持仓/关注股出财报当天自动拉 8-K 跑解读 + 复评。
- 事件流过滤推送：Trump 帖子分级、macro 事件，命中持仓才推。
- 每日自动收盘报告：capital-rotation / 持仓复盘自动落 journal。

### 二、托管服务——「省掉配置和运维」

- 托管模型额度：不配 key 直接可用，平台代付 token，按量计费。
- 数据托管/云同步：见下方专节。
- 手机推送通道：App/PWA push/Telegram bot 的中转服务。与「盘中自动警报」天然捆绑——警报没有送达渠道只值一半钱。

### 三、高级分析——「更深的判断工具」

- 决策复盘统计：基于已有的 journal/decisions/*.json 与违规账单做可视化——胜率归因、违规成本量化。与产品灵魂（纪律）最契合，差异化最强，但需用户先积累决策数据。
- 期权高级面：options-levels 磁铁位/gamma 墙自动刷新并画进图表。
- 多市场深度包：港股/A 股/韩国链路作扩展包。
- bench 报告：私有模型交易基准的结果订阅。

### 四、明确不做

- 信号订阅/荐股：违背仓库纪律（TD-NOISE-01、TD-INTENT-01），且有投资建议合规风险。
- 高频裸波动警报：制造噪音，与产品哲学冲突。

## 专节：数据托管 / 云同步（已定关键决策）

现状：研究报告（journal/、stocks/）、图表数据（journal/charts/data/）、决策记录全在本地单机。

**为什么这个产品适合卖存储**：数据天生是档案性质——只增不改的沉淀物，越积越值钱，迁移成本随时间变高，续订动力自然。研究库浏览免费，「在任何设备上浏览」收费，分界干净，不误伤本地用户。托管一旦有了，「托管部署 web 版」「手机翻研究库」「换机无缝迁移」都是同一能力的副产品。

**已定决策**：

- **同步模型：local-first。** 本地永远是真相源，云端是副本 + 中转。不联网一切照旧，与开源版哲学一致，服务不可用不影响本地使用。
- **计费维度：按量（配额）。** 档位结构设计为可扩展——当前只有一个等级维度（单一档位），后续可在同一结构上叠加更多维度（容量档、设备数等），不重构计费。

**同步范围分期**：

| 期 | 内容 | 理由 |
|---|---|---|
| 第一期 | journal/**/*.md、stocks/*.md、journal/charts/data/*.json、journal/decisions/*.json | 纯文本/JSON、追加型或写后冻结（immutable），文件级同步即可，冲突概率低 |
| 不碰 | SQLite 库（AI 设置、凭证、关注、对话历史） | 二进制库无法文件级同步；凭证（API key）出本机带来安全责任，明确排除 |

**隐私**：交易日志是敏感数据，至少静态加密，目标端到端加密（服务端不可读）——本身可作卖点。

**依赖**：账号体系。Dodo 订阅已搭好「用户-license」骨架，账号可长在其上。

## 优先级建议

1. 盘中自动警报 + 手机推送（捆绑）：付费理由最硬，离现有 scheduler 代码最近，用户感知最强。
2. 托管模型额度 / 数据托管：并列。前者降新用户门槛，后者服务器成本近零但需先有账号体系。
3. 决策复盘统计：等用户积累决策数据后推出。
4. 云同步之上的托管 web 版、多市场包、bench 报告：远期。
