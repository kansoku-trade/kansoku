# K 线形态标注优化：含金量打分 + 确认生命周期

日期：2026-07-06
状态：已通过讨论确认（方向 A + B 为主干，C、D 顺带产出）

## 背景与问题

`intraday` 图表的自动 K 线形态检测（`apps/server/src/services/candlePatterns.ts`）目前把所有检测结果一视同仁地画上图，导致标注没有实际参考价值。典型例子：夜盘 04:35 ET 的「下影针线」——成交量极小的时段里几笔单子就能扎出长下影，属于流动性噪音。

具体缺陷：

1. 夜盘/盘前盘后的稀薄时段照样触发，不看成交量、不看时段。
2. 单根弱形态（针线、十字星）判定门槛低，横盘里遍地都是。
3. 弱信号和强信号（启明星、吞没等多根组合）视觉上平权，噪音稀释了真信号。
4. 提示语写「需下一根确认」，但下一根走完后标注不会更新为「已确认 / 已失效」。
5. 检测和关键位（当日高低点、EMA、FVG、摆动点）完全脱钩，虽然这些数据服务端都已算好。

## 目标

- 少而精：噪音形态不上图或降级为小点。
- 可决策：每个形态给出明确的确认价 / 失效价，状态自动更新。
- 可信度：附带该形态在本票本周期的历史走对比例（轻量统计，非严肃回测）。

## 方案总览

检测出的形态经过一个新的打分与状态追踪层（`patternScoring.ts`），再按分档渲染：

```
detectCandlePatterns → scorePatterns(打分 + 状态判定 + 历史统计) → 分档渲染 marker
```

### 1. 含金量分数（0–100）

`score = clamp(底分 + 时段调整 + 成交量调整 + 位置调整, 0, 100)`

**底分（按形态强度）**
- 强多根组合（吞没、启明星/黄昏星、红三兵/三只乌鸦，即 meta 中 `strong: true`）：55
- 弱单根/双根形态（针线、锤子、孕线、乌云盖顶、镊子、光头大阳等）：30
- 中性十字星类：20

**时段调整**（用现有 `classifySession`）
- regular：+10
- pre / post：-10
- overnight：弱形态与中性形态**直接丢弃**（不进入输出）；强形态 -25

**成交量调整**（信号 K 线的量 / 近 20 根均量；多根形态取组合内最大值）
- ≥ 1.5 倍：+15
- ≥ 1.0 倍：+5
- < 0.7 倍：-15
- 其余：0

**位置调整**（距离以近 14 根平均波幅 avgRange 为尺度）
- 看涨形态的最低点、或看跌形态的最高点，落在任一关键位 0.5×avgRange 范围内：+15
- 关键位集合：摆动高低点（`findSwings` 结果）、各条 EMA 当根数值、FVG 缺口上下沿
- 不贴关键位：0

**渲染分档**
- score ≥ 65：完整箭头标注
- 45 ≤ score < 65：小圆点（悬停看详情）
- score < 45：不上图（数据保留，`summary.candle_patterns` 仍带分数喂给 AI）

### 2. 确认生命周期

每个有方向的形态在检测时输出两个价位：

- `confirm_price`：看涨 = 信号 K 线实体上沿（多根组合取最后一根的最高价）；看跌反之
- `invalidate_price`：看涨 = 形态最低点；看跌 = 形态最高点

状态机（观察窗口 = 形态之后 3 根 K 线，按收盘价判定）：

- `pending` 待确认：窗口内还没有 K 线收出结果（空心标注 + 「?」）
- `confirmed` 已确认：某根收盘价越过 confirm_price（实心标注 + 「✓」）
- `invalidated` 已失效：某根收盘价跌破/涨破 invalidate_price（变灰缩成小点 + 「✗」）
- `expired` 已过期：3 根走完两边都没触发（变灰缩成小点）

中性形态（十字星类）无方向，不参与状态机（`status` 为 null）。

提示框（tooltip）内容改为直接可用：形态名、状态、**确认价 $X ｜ 失效价 $Y**、含金量分数、原有形态含义、历史统计。

### 3. 历史走对比例（轻量统计）

检测本来就跑全量历史，顺带统计：本票本周期上，每种形态**已确认**之后的表现——

- 走对定义：确认那根 K 线之后 5 根内，任一收盘价较确认时收盘价朝 bias 方向移动 ≥ 0.5×avgRange
- 样本 ≥ 8 次才在 tooltip 显示（如「近 12 次确认后 9 次走对」），不足显示「历史样本不足」

这不是严肃回测，只是给「该不该信这个标注」一个粗略参照。

## 改动清单

| 文件 | 改动 |
|---|---|
| `packages/shared/types.ts` | `CandlePattern` 加 `score`、`status`、`confirm_price`、`invalidate_price`、`stats` 字段 |
| `apps/server/src/services/candlePatterns.ts` | 检测输出附带 confirm/invalidate 价位与形态跨度（span） |
| `apps/server/src/services/patternScoring.ts`（新增） | 打分、状态机、历史统计 |
| `apps/server/src/services/intraday.ts` | `coerceIntradayTimeframe` 里接入打分层（FVG 检测提前到打分之前）；marker 构建按分档 + 状态渲染，tooltip 加确认价/失效价/分数/统计 |
| `apps/server/test/candlePatterns.test.ts` | 补确认价/失效价断言 |
| `apps/server/test/patternScoring.test.ts`（新增） | 时段丢弃、量能加减分、位置加分、状态流转、统计口径 |
| `apps/server/test/intraday.test.ts` | 分档渲染与 tooltip 相关断言更新 |

web 端无需改动：形态标注全部由服务端生成 marker，前端只负责渲染。

## 不做的事

- 不做严肃回测框架（滑点、多周期汇总、显著性检验）。
- 不改 AI 手动标注信号（`signals`）的链路，只改自动检测形态。
- 不新增用户可调参数（阈值先内置，用出感觉再说）。
